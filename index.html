Principais Comandos que aprendemos na Aula.

Git Version: Utilizado para verificar qual a versão do Git está instalado na maquina.

Configurando o git pelo terminal:

Precisamos inserir um usuário para saber quem fez determinadas alterações no código.

inserindo usuário: git config --global user.name "Nome Sobrenome"

inserindo e-mail: git config --global email "e-mail"

Para verificar se está correto, basta usar os comandos abaixo:

git config user.name

git config email

Mais comandos

Git Init: Utilizamos para criar um novo repositório "Pasta" Local

Dir: Utilizado para verificar quais os arquivos temos em um repositório

CD + nome repositório: utilizado para acessar determinado repositório

CD..: Utilizado para acessar o repositório anterior, ou seja saímos da subpasta que estamos para um nível acima

Code .: Acessamos através do terminal o repositório que iremos trabalhar e utilizamos esse comando para abrir dentro do VS code.

Git Status: Utilizado para verificar o status atual do arquivo no repositório local.

Git Add: Adiciona os novos arquivo e alterações realizadas nos códigos para staging área.

Git RM - Caso tenha incluído algum arquivo na staging área que não queira fazer o commit, pode excluir utilizando esse comando.

Git Commit: Salva as mudanças realizadas em um repositório.

Git Log: Utilizado para verificar o historio de alterações em um repositório.

Git log --online: Utilizado para verificar o histórico de forma resumida.

Git remote add orgin "Link GitHub": Utilizado para determinar qual o destino do Origin

Git push: subir o repositório local para o GitHub

Git Pull: Utilizado para puxar o arquivo do GitHub para o diretório local

Git Clone: Utilizado para clonar(Criar uma cópia) o diretório do GitHub
